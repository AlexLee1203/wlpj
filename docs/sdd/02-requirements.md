# 02 Requirements

## 背景

- 使用者希望透過規律運動達成減重或維持健康生活，但常常缺少一個簡單工具來安排與記錄每日運動。
- 由於不同使用者偏好的運動類型不同，第一版系統不負責自動推薦計劃，而是先提供手動建立與追蹤運動項目的能力。
- 系統應幫助使用者清楚記錄運動項目、持續時間、熱量消耗與完成狀態。

## 功能需求

- `FR-1`：使用者可以新增一筆運動紀錄。
- `FR-2`：每筆運動紀錄至少包含運動項目名稱、持續時間、預估消耗熱量、是否完成。
- `FR-3`：使用者可以查看目前已建立的運動紀錄列表。
- `FR-4`：使用者可以修改某筆運動紀錄的完成狀態。
- `FR-5`：系統應顯示每日所有運動的總消耗熱量。

## 非功能需求

- `NFR-1`：效能
- 在一般個人使用情境下，新增與讀取紀錄應快速完成，不應有明顯等待。

- `NFR-2`：安全性
- 第一版不含登入與雲端同步，但仍需對輸入欄位做基本驗證，避免空值與不合理數值。

- `NFR-3`：可維護性
- 專案程式碼應依功能模組拆分，方便後續加入圖表、帳號或推薦功能。

## 使用者故事

- 身為想減重的使用者，我想要記錄每天做了哪些運動，以便追蹤自己是否有持續執行計劃。
- 身為想維持健康生活的使用者，我想要記錄每個運動的持續時間與熱量消耗，以便了解每天的活動成果。
- 身為使用者，我想要標記運動是否完成，以便快速知道今天的計劃完成度。

## 驗收條件

- [ ] 使用者可以建立一筆包含運動名稱、持續時間、消耗熱量的紀錄。
- [ ] 使用者可以在列表中看到所有已新增的運動紀錄。
- [ ] 使用者可以將某筆運動標記為已完成或未完成。
- [ ] 系統可以正確計算並顯示當日總消耗熱量。
- [ ] 當持續時間或消耗熱量為空值、負數或非數字時，系統應阻止儲存並提示錯誤。

## 風險與假設

- 風險：若一開始就加入自動推薦運動計劃，需求與規格會快速膨脹。
- 風險：熱量消耗計算若依不同體重、強度、年齡精算，第一版會變得複雜。
- 假設：第一版中的消耗熱量由使用者手動輸入，暫不由系統自動計算。
- 假設：第一版先以單機或本地資料為主，不處理跨裝置同步。
