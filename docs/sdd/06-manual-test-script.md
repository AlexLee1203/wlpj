# 06 Manual Test Script

## 目的

- 用固定步驟完成 `v1.0.0` 的手動驗收。
- 驗證桌面操作、手機尺寸、PWA 安裝、離線啟動、備份還原流程。

## 測試前準備

1. 在專案目錄執行：

```bash
npm run dev
```

2. 用瀏覽器開啟終端顯示的本機網址。
3. 若瀏覽器中已有舊資料，先匯出備份，避免覆蓋。

## Case 1：桌面基本流程

### 目標

- 驗證新增、編輯、刪除、切換日期都正常。

### 步驟

1. 開啟首頁。
2. 在表單輸入：
   - 運動項目：`快走`
   - 持續時間：`30`
   - 消耗熱量：`220`
3. 送出表單。
4. 確認清單出現 `快走`，統計區顯示總熱量增加。
5. 點擊 `標記為已完成`。
6. 確認完成數與完成率更新。
7. 點擊 `編輯`。
8. 把運動項目改成 `慢跑`，把熱量改成 `300`。
9. 送出更新。
10. 確認清單改為 `慢跑`，熱量更新。
11. 點擊日期切換到前一天。
12. 確認前一天清單是空的或顯示該日期既有資料。
13. 切回今天。
14. 點擊 `刪除`。
15. 在確認視窗按確認。
16. 確認紀錄已消失。

### 預期結果

- 新增成功。
- 編輯成功。
- 刪除前有確認提示。
- 切換日期後只顯示該日期資料。
- 統計區數值會即時更新。

## Case 2：手機尺寸排版

### 目標

- 驗證小螢幕下表單、預設卡片、備份區仍可操作。

### 步驟

1. 開啟瀏覽器開發者工具。
2. 切換到手機尺寸，例如 `iPhone 12`。
3. 檢查以下區塊是否可完整看到：
   - 頂部標題
   - 統計區
   - 日期切換區
   - 快速新增預設運動
   - 備份與整理
   - 運動表單
   - 運動清單
4. 點擊任一預設運動卡片。
5. 確認表單有正確填入值。
6. 在手機尺寸下新增一筆資料。
7. 在手機尺寸下點擊編輯、刪除、切換完成。

### 預期結果

- 沒有內容被切掉。
- 按鈕可點擊。
- 預設卡片會改成單欄顯示。
- 表單與清單操作正常。

## Case 3：PWA 安裝到主畫面

### 目標

- 驗證可安裝並以獨立 App 形式開啟。

### 步驟

1. 在支援 PWA 的瀏覽器開啟網站。
2. 使用瀏覽器的 `Install App` / `加入主畫面` 功能。
3. 完成安裝。
4. 從桌面或主畫面開啟 App。

### 預期結果

- 可成功安裝。
- 開啟後是獨立視窗或 App 介面，不是一般瀏覽器分頁樣式。
- App 名稱顯示為 `減重日誌` 或 `減重計劃日誌`。

## Case 4：離線重新開啟

### 目標

- 驗證 PWA 快取可支援基本離線啟動。

### 步驟

1. 先在線狀態開啟 App 一次。
2. 關閉所有該 App 視窗。
3. 中斷網路，或在 DevTools 模擬 `Offline`。
4. 重新開啟 App。

### 預期結果

- App 基本畫面可載入。
- 已儲存的本地資料仍可顯示。
- 若有不支援的操作，不應造成整頁空白。

## Case 5：備份匯出與匯入

### 目標

- 驗證資料可以被匯出並正確還原。

### 步驟

1. 先建立至少兩筆不同日期的運動紀錄。
2. 點擊 `匯出全部紀錄`。
3. 確認瀏覽器下載一個 `.json` 檔案。
4. 點擊 `清空當日紀錄`，確認當日資料被清掉。
5. 再使用 `匯入備份檔` 選擇剛下載的 JSON。
6. 檢查今天與其他日期資料是否恢復。
7. 再嘗試匯入一個錯誤格式檔案。

### 預期結果

- 匯出成功下載 JSON。
- 匯入後資料恢復。
- 匯入錯誤檔案時，畫面顯示錯誤訊息。

## 測試完成後

- 將結果回填到 [05-release-checklist.md](./05-release-checklist.md)。
- 若有失敗項目，記錄重現步驟、實際結果、預期結果。
