# 03 Spec

## 系統概觀

- 這是一個可安裝為 PWA 的 Web 應用，讓使用者建立每日運動紀錄、追蹤完成狀態，並查看當日總消耗熱量。

## 架構決策

- 前端：Web 單頁應用（SPA），提供表單輸入、紀錄列表、每日熱量總計。
- 後端：第一版不使用遠端後端服務，先以前端本地資料儲存為主。
- 資料庫：瀏覽器本地儲存，優先使用 `localStorage` 作為第一版方案；若之後資料模型變複雜，可升級到 `IndexedDB`。
- 部署方式：靜態網站部署，可放在 Vercel、Netlify 或 GitHub Pages。
- PWA：提供 `manifest` 與 `service worker`，支援加入主畫面與基本離線載入。

## 模組切分

- `運動紀錄表單`：新增運動名稱、持續時間、消耗熱量。
- `運動紀錄列表`：顯示所有紀錄與完成狀態。
- `每日統計區塊`：顯示當日總熱量與完成情況。
- `日期切換模組`：切換查看不同日期的運動紀錄。
- `預設運動模組`：提供快速填入的常用運動模板。
- `備份模組`：支援資料匯出、匯入與清空當日紀錄。
- `本地儲存模組`：負責讀寫瀏覽器本地資料。
- `PWA 模組`：註冊 service worker、管理 manifest 與快取策略。

## 資料模型

- `ExerciseLog`
- `id`：字串，唯一識別值。
- `name`：字串，運動名稱。
- `durationMinutes`：數字，持續時間（分鐘）。
- `caloriesBurned`：數字，預估消耗熱量。
- `completed`：布林值，是否完成。
- `date`：字串，紀錄所屬日期，格式可先用 `YYYY-MM-DD`。

## API / 介面

- 第一版不設計遠端 API。
- 前端內部介面：
- `createExerciseLog(input)`：建立一筆運動紀錄。
- `listExerciseLogsByDate(date)`：查詢指定日期紀錄。
- `toggleExerciseLogCompleted(id)`：切換完成狀態。
- `updateExerciseLog(id, input)`：更新既有運動紀錄。
- `removeExerciseLog(id)`：刪除單筆運動紀錄。
- `removeLogsByDate(date)`：清空某一天的全部紀錄。
- `getDailyCalories(date)`：計算指定日期總熱量。
- `exportLogs()`：匯出全部資料。
- `importLogs(file)`：從備份檔還原資料。

## 狀態流程

- 使用者輸入運動資料
- 前端驗證欄位是否合法
- 儲存到本地資料
- 更新列表與每日總熱量
- 使用者可再切換完成狀態
- 使用者可切換日期查看過去或未來計劃
- 使用者可匯出或匯入本地備份
- 畫面即時重新渲染結果

## 錯誤處理

- 運動名稱為空時，不允許儲存並顯示提示。
- 持續時間不是正整數時，不允許儲存並顯示提示。
- 消耗熱量不是 0 或正數時，不允許儲存並顯示提示。
- 本地儲存失敗時，顯示一般性錯誤訊息。
- 匯入非合法 JSON 備份時，顯示匯入失敗訊息。
- 刪除單筆紀錄或清空當日紀錄前，需先確認。
- 若離線狀態下重新整理頁面，PWA 應仍能載入已快取的基本畫面資源。

## 驗證方式

- 單元測試：
- 驗證資料處理函式，例如新增紀錄、切換完成狀態、計算每日熱量、排序、匯入匯出解析。

- 整合測試：
- 驗證表單送出後，列表與統計是否同步更新。
- 驗證匯入資料後，畫面是否正確反映新資料。

- 手動驗證：
- 安裝 PWA 到桌面或手機主畫面。
- 關閉網路後重新開啟 App，確認基本頁面仍能載入。
- 新增一筆運動後，確認總熱量正確更新。
- 匯出備份後，重新匯入確認資料恢復正確。
